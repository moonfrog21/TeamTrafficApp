# Implementation Plan: Google Maps & Routing Update

## Context and Workspace State
**Target Environment:** VS Code with Flutter extension active.
**Objective:** Implement the updates detailed in `UpdatePRD.md` by transitioning the UI to a Google Maps interface, adding destination routing, and simplifying the core telemetry controls, while maintaining the background GPS transmission.
**Execution Strategy:** Execute these steps sequentially. Do not proceed to the next step until the current one is verified and functional.

---

## Step 1: Install Dependencies & Configure Native Environments
- **Flutter Setup:** Add `google_maps_flutter`, `flutter_polyline_points`, and `uuid` to `pubspec.yaml` under dependencies. Note that `http` and `geolocator` are already present. Run `flutter pub get`.
- **Android Configuration:** Navigate to `android/app/build.gradle.kts` (this project uses Kotlin DSL, not Groovy) and change `minSdk = flutter.minSdkVersion` to `minSdk = 21` (required by `google_maps_flutter`). Add the Google Maps API metadata tag inside the `<application>` block of `android/app/src/main/AndroidManifest.xml` (leave a placeholder for the API key).
- **iOS Configuration:** Navigate to `ios/Runner/AppDelegate.swift`, add `import GoogleMaps` at the top, and inject `GMSServices.provideAPIKey("YOUR_API_KEY")` inside the `application(_:didFinishLaunchingWithOptions:)` method before the `return` statement. Note: this project does not have a Podfile yet; one will be auto-generated by Flutter when iOS dependencies are resolved via `flutter pub get` or `flutter build ios`.

## Step 2: Inject the Google Cloud API Key
- Obtain a valid Google Cloud API Key with Maps SDK (Android/iOS), Places API, Geocoding API, and Routes/Directions API enabled.
- **iOS:** Replace the placeholder `"YOUR_API_KEY"` in `ios/Runner/AppDelegate.swift`.
- **Android:** Replace the placeholder in `android/app/src/main/AndroidManifest.xml` within the `<meta-data android:name="com.google.android.geo.API_KEY" android:value="..." />` tag.
- **Flutter:** Create a `lib/config.dart` or similar constant file to securely store the API key for making HTTP requests to the Places and Routes APIs on the Dart side.

## Step 3: Revamp the Main Screen
- Refactor the UI in `lib/main.dart` (specifically the `HomeScreen`).
- Replace the main layout with a `Stack` widget.
- Insert a `GoogleMap` widget from `google_maps_flutter` as the base layer of the `Stack` to render as the fullscreen background.
- Set an initial fallback `CameraPosition`.
- Position the existing green/red "Start/Stop GPS Data Transmit" button at the bottom center of the screen, overlaid on the map.
- Visually scale down the start/stop button and **remove all instructional text** from the screen.
- Verify the map renders correctly and the button successfully toggles the background telemetry without errors.

## Step 4: Implement Dynamic Camera Tracking
- Retrieve the `GoogleMapController` from the map's `onMapCreated` callback.
- Modify the existing location logic (or create a new location stream) so that whenever the user's GPS coordinates update, the map's camera position automatically shifts (`controller.animateCamera`) to keep the user centered on the screen.

## Step 5: Implement Destination Search Bar & Places Autocomplete
- Add a search bar (`TextField`) overlaid at the top of the map.
- Implement an autocomplete dropdown below the search bar.
- On text input, execute an HTTP GET request to the **Google Places Autocomplete API**, attaching a generated session token (via the `uuid` package) and your API key. Display the suggestions.
- Upon selecting a suggestion, execute an HTTP GET request to the **Google Geocoding API** (or Place Details API) to resolve the Place ID/address into exact latitude and longitude coordinates.
- Place a map marker (`Marker`) at these destination coordinates.

## Step 6: Route Calculation & Polyline Rendering
- Create a routing service to query the **Google Directions API** (legacy, via HTTP GET) or the **Google Routes API** (newer, via HTTP POST with JSON body).
- Pass the user's current GPS location as the `origin` and the geocoded coordinates from Step 5 as the `destination`.
- Parse the JSON response to extract the `overview_polyline` string.
- Utilize the `flutter_polyline_points` package to decode the polyline string into a list of `LatLng` objects.
- Feed these coordinates into the `GoogleMap` widget as a `Polyline` object, rendering a distinct path (e.g., a blue line with a width of 5) on the screen.

## Step 7: Final Integration & Cleanup
- Ensure the settings cog icon remains visible and functional (e.g., wrap it in a floating circular button at the top right) to configure the ngrok URL.
- Test the core telemetry functionality to verify that `user`, `latitude`, `longitude`, and `time` are still correctly POSTed to ngrok every 5 seconds while tracking and routing are actively running.
- Implement robust error handling (e.g., location permissions denied, invalid API keys, network failures during Place searches).
